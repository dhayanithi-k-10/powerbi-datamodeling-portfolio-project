# Power BI Data Modeling Portfolio Project

## Overview

This project demonstrates the transformation of a complex operational dataset into a clean, scalable, and analytics-ready data model using Microsoft Power BI.

The original dataset contained multiple transactional tables with complex relationships and structures that were not suitable for business reporting. Using dimensional modeling principles, the data was redesigned into a Star Schema with optimized fact and dimension tables, creating a semantic model that supports efficient analysis and future dashboard development.

---

## Project Objectives

- Transform raw operational data into an analytical data model
- Design a Star Schema following dimensional modeling principles
- Optimize table relationships for reporting
- Build reusable DAX measures
- Create a scalable semantic model
- Implement Row-Level Security (RLS)

---

## Problem Statement

The source dataset consisted of multiple operational tables representing different business processes, including:

- Orders
- Customers
- Products
- Inventory
- Campaigns
- Shipments
- Invoices
- Payments

The data model contained:

- Complex relationships
- Redundant tables
- Inconsistent structures
- Poor readability
- Limited support for analytical reporting

---

## Solution

The dataset was redesigned into a dimensional model by:

- Creating Fact and Dimension tables
- Designing a Star Schema
- Optimizing relationships
- Implementing a dedicated Date dimension
- Organizing reusable DAX measures
- Applying Row-Level Security (RLS)

The resulting semantic model is clean, scalable, and optimized for business intelligence reporting.
